## What Are These Templates?

These are **ready-to-use ABC notation templates** for common programming journey patterns. Each template provides:

1. **Complete musical structure** - Pre-composed sections for each phase
2. **Customization points** - Clear markers where to add your specific details
3. **Usage guidelines** - Instructions for adapting the template
4. **Musical theory notes** - Explanations of why certain choices were made

## Available Templates

### 1. `bug_discovery.abc`
**Use When:** A bug is discovered and needs fixing

**Structure:**
- Part A: Confident Implementation (before bug)
- Part B: The Realization (bug discovered - dissonance!)
- Part C: Investigation (searching for cause)
- Part D: Breakthrough (solution found)
- Part E: Resolution (tests passing, bug fixed)

**Key Features:**
- Diminished chords for confusion
- Chromatic movement for searching
- Modulation A minor → C major (problem → solution)

**Best For:** Issues #4 style bug discoveries

## How to Use Templates

### Step 1: Choose the Right Template

Match your code journey to a template:
- **Bug found?** → `bug_discovery.abc`
- **Feature completed?** → `feature_complete.abc`
- **PR merged?** → `pr_merge.abc`
- **Code too slow?** → `performance_optimization.abc`
- **Unexpected problem?** → `crisis_recovery.abc`

### Step 2: Copy and Customize

1. **Copy template file** to your session folder
2. **Update header:**
   ```abc
   X:1
   T:Your Specific Title Here
   C:JamAI (G.Music Assembly)
   ```

3. **Replace [ADD COMMENT] markers** with actual technical details:
   ```abc
   % [ADD COMMENT: What bug was discovered?]
   % ↓ Replace with:
   % Bug discovered: metadata written to wrong note (session_id mismatch)
   ```

4. **Adjust musical parameters:**
   - **Tempo**: Slower for serious issues, faster for exciting wins
   - **Dynamics**: pp for confusion, fff for Jerry's joy
   - **Key**: Minor for problems, major for solutions

5. **Add lyrics (optional):**
   ```abc
   "G"G4 B2 d2 g2 |
   w: Found the bug in the code base
   ```

### Step 3: Extend or Simplify

**To Extend:**
- Add more movements for complex journeys
- Insert Jerry's joy section for major wins:
  ```abc
  [Q:1/4=180]"fff"
  "G"[GBdg]2[GBdg]2 [GBdg]2[GBdg]2 |
  w: JER-RY'S JOY! VIC-TOR-Y!
  ```
- Add Assembly collaboration sections

**To Simplify:**
- Remove unnecessary sections
- Combine movements
- Shorten repeats

## Combining Templates

You can combine multiple templates for complex sessions:

### Example: Bug During Feature Implementation

1. Start with `feature_complete.abc` Part A-B (vision + implementation)
2. Insert `bug_discovery.abc` Part B-D (bug found + fixed)
3. Continue with `feature_complete.abc` Part C-D (testing + completion)

### Example: Performance Crisis Then Optimization

1. Start with `crisis_recovery.abc` Part A-C (success → crisis → investigation)
2. Insert `performance_optimization.abc` Part C-D (optimization + victory)
3. Finish with `crisis_recovery.abc` Part F (peaceful resolution)

## Creating Custom Templates

Want to create your own template?

1. **Study existing templates** - See patterns
2. **Identify the narrative arc** - What's the journey?
3. **Choose appropriate keys** - Minor for problems, major for solutions
4. **Plan tempo progression** - Match emotional intensity
5. **Add clear customization points** - `[ADD COMMENT]` markers
6. **Document usage** - Explain when/how to use
7. **Save in `/templates/` folder** - Make reusable!

## Best Practices

✅ **DO:**
- Add specific technical details in comments
- Adjust tempo/dynamics to match your experience
- Use lyrics for memorable moments
- Keep Jerry's joy sections (fff) when appropriate
- Test your composition with an ABC player

❌ **DON'T:**
- Keep generic [ADD COMMENT] markers in final version
- Use wrong key for emotion (major for bugs, minor for solutions)
- Ignore tempo/dynamic markings - they encode meaning!
- Forget to update the title with your specific issue/feature
- Rush through - musical composition takes thought!
