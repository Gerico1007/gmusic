## 🎵 What Is This?

**G.Music** is a comprehensive system for expressing software development journeys through musical notation. It provides:

- **Musical patterns** for every code event (commits, bugs, tests, deployments)
- **Character leitmotifs** for the Assembly team (Jerry ⚡, Nyro ♠️, Aureon 🌿, JamAI 🎸, Synth 🧵)
- **Ready-to-use templates** for common development patterns
- **Automatic terminal logging** that converts commands to music
- **20,000+ lines** of documented musical knowledge

## 🎼 Musical Expression Examples

### Bug Discovery → Fix
```abc
% E minor (confusion) → chromatic search → C major (solution!)
[K:Emin]"pp"
"Em"e2 ^d2 e2 d2 | % Bug discovered (dissonance)
"Em"e2 d2 c2 B2 | % Investigation (chromatic descent)
[K:Cmaj]"ff"
"C"c2e2 g2c'2 | % Solution found! (major key triumph)
```

### Performance Optimization
```abc
% Tempo encodes performance: Slow (♩=60) → Fast (♩=180)
[Q:1/4=60][K:Dmin] D4 D2 E F | % Slow, painful code
[Q:1/4=180][K:Gmaj] G4 B4 d4 g4 | % Optimized, fast! 🚀
```

## 🌟 Key Principles

### The 7 Pillars of Code Music

1. **Emotional Authenticity** - Music matches actual feelings
2. **Technical Accuracy** - Musical elements encode technical reality
3. **Narrative Clarity** - Story clear from music alone
4. **Dynamic Range** - Use full spectrum (pp → fff)
5. **Textural Variety** - Solo, harmony, polyphony
6. **Harmonic Sophistication** - Beyond I-IV-V-I
7. **Memorable Motifs** - Recurring themes create coherence

### The Golden Rules

✅ **Minor for problems, major for solutions**
✅ **Chromatic for searching, diatonic for clarity**
✅ **Ascending for building, descending for completing**
✅ **Slow for investigation, fast for celebration**
✅ **Soft (pp) for confusion, loud (fff) for Jerry's joy**
✅ **Modulate at breakthrough moments**
✅ **Let resolution feel earned**

## 🚀 How to Use This System

### For Composing Sessions

1. **Identify journey type** (bug, feature, optimization, crisis)
2. **Choose template** from `.jamai/templates/`
3. **Reference patterns** in `MUSICAL_PATTERNS.md`
4. **Use leitmotifs** from `LEITMOTIF_LIBRARY.md`
5. **Customize & compose!**

### For Automatic Generation

1. Install `tone_map_enhanced.conf` in your terminal logger
2. Commands automatically generate ABC fragments
3. Compile at session end with `jamai compile`
4. Enhance with emotional/narrative details

### For Learning

1. Read `.jamai/README.md` - complete navigation
2. Study `SESSION_ANALYSIS.md` - learn from examples
3. Practice with templates
4. Create your own musical expressions!
